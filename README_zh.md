# SkillForge

> **给我一个意图，还你一个完整 Skill。**

七步元认知推导框架——不是模板库，不是规则集，而是七种思维方式。任何意图经过这七步，都能长出完整的、可执行的 Skill。

---

## 七步总览

```
意图
  │
  ├─ 搜：网上有没有类似的？学工具/流程/异常/阈值
  │    ↓ 学到了 → 调整工具/流程/异常/阈值
  │    ↓ 没搜到 → 确认空白，从零推导
  │
  ├─ 分：阴阳（做什么 / 不做什么）
  │    ↓
  ├─ 叠：四力（工具 / 任务 / 判断 / 门控）
  │    ↓
  ├─ 展：八卦（输入 / 输出 / 起势 / 流向 / 异常 / 标准 / 边界 / 握手）
  │    ↓
  ├─ 衡：五行（金→木→火→水→土）传导链检验
  │    ↓
  ├─ 验：断点回头补（发现断裂回到对应步骤）
  │    ↓
  └─ 成：填完即止
```

---

## 推导示例

**意图：** "我想做一个竞品价格监控"

| 步骤 | 输出 |
|------|------|
| **搜** | 学到：采集→对比→告警流程，阈值用 10% 变化率 |
| **分** | 做：定期采集/记录变化/触发告警/生成日报。不做：不改价/不预测/不跨品类 |
| **叠** | 工具：4个 · 任务：5步 · 判断：3个量化阈值 · 门控：5条 |
| **展** | 输入：竞品型号列表 + 上次价格记录。输出：`daily_report.json` + `price_alert.md` |
| **衡** | 木→火→土→金→水，传导链完整，五行无断裂 |
| **验** | 扩展预设已定义：无阻塞/有方向/阈值可调 |
| **成** | **完整 Skill ✓** |

## 五行：不是标签，是五种力量

五行之间有相生有相克，构成动态推理链条：

**相生（力量从哪来）：**
- 木 → 火：有方向才有判断
- 火 → 土：有判断才有沉淀
- 土 → 金：有沉淀才有结构
- 金 → 水：有结构才有流动
- 水 → 木：有流动才有新方向

**相克（力量往哪去）：**
- 金 → 木：结构修剪过度发散的方向
- 木 → 土：创新突破过度稳定的沉淀
- 土 → 水：沉淀阻塞流动
- 水 → 火：流动平衡过热的判断
- 火 → 金：判断瓦解过度僵化的结构

**正确提问方式：**
```
不是问："工具够吗？任务有序吗？"

而是问：
  方向是否过度发散（木被金克过度）？
  判断是否过度僵化（火被水克过度）？
  产出是否阻塞流动（土克水过度）？
```

## 五种入口

| 入口 | 适用场景 |
|------|---------|
| **链路节点** | Skill 属于某个链路的一环（采集→清洗→分析→洞察） |
| **独立领域** | Skill 有自己的上下游，不属于任何链路 |
| **战略链路** | Skill 在 L1（理解）/ L2（改变）/ L3（守住） |
| **分析洞察** | Skill 有统计层（是什么）+ 洞察层（为什么） |
| **5分钟速成** | <10工具、<5步、独立使用，追求能用就行 |

## 核心原则

- **不是模板库**：不给预设类型，给思维方式
- **不是规则集**：不给工具清单，给推理路径
- **扩展预设**：每个 Skill 必须预设扩展方向，让 Skill 从封闭变开放架构
