# SkillForge

> **给我一个意图，还你一个完整 Skill。**

七步元认知推导框架——不是模板库，不是规则集，而是七种**思维方式**。任何意图经过这七步，都能长出完整的、可执行的 Skill。

```yaml
输入:  一句话意图
输出:  完整可运行的 Skill
耗时:  15-30 分钟
```

---

## 30 秒看懂

**痛点** — 你有个 AI Skill 的想法，但不知道：
- 该用哪些工具
- 任务按什么顺序跑
- 这个 Skill 在你的链路里是什么位置
- 它**绝对不能**做什么
- 怎么判断它跑得好不好

**解法** — 回答 7 个问题，产出完整 Skill：
- **3-10 个工具**（它需要什么家伙）
- **5-10 步任务**（按什么顺序做）
- **2-5 个量化判断**（带阈值，不是凭感觉）
- **3-5 条门控**（怎么算做完了）
- **3+ 条禁止规则**（什么绝不能做）
- **3+ 个扩展方向**（以后怎么长大）

## 五行：不是标签，是五种力量

五行是**动态推理系统**——不是检查清单。问的是：**哪些力量在场？哪个弱/过旺？**

```
相生 — 力量从哪来：
  木 → 火    方向给判断注入目的
  火 → 土    判断产出沉淀
  土 → 金    沉淀塑造结构
  金 → 水    结构开启流动
  水 → 木    流动生长新方向

相克 — 力量往哪去：
  金 → 木    结构修剪过度发散的方向
  木 → 土    创新突破过度稳定的沉淀
  土 → 水    沉淀阻塞流动
  水 → 火    流动平衡过热的判断
  火 → 金    决策重塑过度僵化的结构
```

**错误提问**："工具够吗？任务有序吗？"
**正确提问**："方向是否过度发散？判断是否过度僵化？产出是否阻塞流动？"

## 术语速查（Plain English）

skillforge 借用了易经八卦 + 五行的意象——不是玄学，是**紧凑的推理模式标签**。它们真正代表什么：

| 中文 | 直译 | Plain English |
|------|------|---------------|
| **太极** | 混沌 | 你开始的种子意图 |
| **阴阳** | 阴-阳 | **做什么 / 绝不能做什么** |
| **四象** | 四种形象 | **工具 / 任务 / 判断 / 门控** |
| **乾** (天) | 刚健 | **输入** — Skill 接收什么数据 |
| **坤** (地) | 柔顺 | **输出** — Skill 产出什么文件/消息 |
| **震** (雷) | 启动 | **起势** — 第一个任务 |
| **巽** (风) | 渗透 | **流向** — 产出送到哪个下游 |
| **坎** (水) | 险陷 | **异常** — 出错时怎么办 |
| **离** (火) | 附着 | **成功标准** — 怎么算做完了 |
| **艮** (山) | 止息 | **边界** — 绝对不能做的事 |
| **兑** (泽) | 愉悦 | **握手** — 跟下游 Skill 的协议 |
| **木** | — | **方向** — Skill 朝哪走 |
| **火** | — | **判断** — 决策力 |
| **土** | — | **产出** — 结果 |
| **金** | — | **结构** — 工具和边界 |
| **水** | — | **流动** — Skill 之间的连接 |

**记不住全部没关系，记住两条：**

> **八卦** = Skill 在世界里的位置（输入/输出/流向/异常……）
> **五行** = Skill 内部的五种力量（方向/判断/产出/结构/流动）

## 七步速查

| # | 操作 | 问什么 | 输出 |
|---|------|--------|------|
| 0 | **搜** | 网上有没有类似的？ | 工具候选 / 流程参考 / 踩坑 / 阈值 |
| 1 | **分** | 做什么 / 不做什么？ | 阴（禁止）+ 阳（要做） |
| 2 | **叠** | 需要什么力量？ | 工具 / 任务 / 判断 / 门控 |
| 3 | **展** | 它在哪？ | 八卦（输入/输出/流向/异常……） |
| 4 | **衡** | 传导链断了吗？ | 五行推理 |
| 5 | **验** | 哪里断？ | 回到第 2/3/4 步补断点 |
| 6 | **成** | 完了吗？ | 完整 Skill + 扩展预设 |

## 五种入口

按你的场景选入口：

| 入口 | 适用场景 |
|------|---------|
| **链路节点** | Skill 属于某个链路的一环（采集→清洗→分析→洞察） |
| **独立领域** | Skill 有自己的上下游，不属于任何链路 |
| **战略链路** | Skill 在 L1（理解）/ L2（改变）/ L3（守住） |
| **分析洞察** | Skill 有统计层（是什么）+ 洞察层（为什么） |
| **5分钟速成** | <10 工具、<5 步、独立使用、追求能用就行 |

见 [RECIPES.md](RECIPES.md) 获取完整入口指南（含每步的具体提问）。

## 核心原则

- **不是模板库**：不给预设类型，给思维方式
- **不是规则集**：不给工具清单，给推理路径
- **扩展预设**：每个 Skill 必须预设扩展方向，让 Skill 从封闭变开放架构
