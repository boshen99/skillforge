# 原理 — 七步元认知，从任意意图推导 Skill

> **核心**：不是规则库，是元认知操作集
> **七个操作**：搜、分、叠、展、衡、验、成
> **任何意图经过这七步，都能长出完整 Skill**

---

## 元认知操作集

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
搜  搜现有     先看网上有没有类似的可以学习
分  切阴阳     把意图切成"做"和"不做"
叠  阴阳叠     阴阳叠出四力（工具/任务/判断/门控）
展  四力展     四力展开八卦（输入/输出/起势/流向/异常/标准/边界/握手）
衡  五行衡     五行相生相克，检验传导链路不断
验  断点补     发现断裂就回头补
成  填完止     能填的填完，不能填的标记待定
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

这七步，不是七条规则，而是七种思维方式。
每一步都可以独立使用，填多填少由你决定。

## 零、搜：搜现有 Skill

```
太极 → 搜

在开始推导之前，先问：
  网上有没有做类似事情的 Skill？

搜什么：
  □ GitHub — Claude Code Skills / 其他 AI Agent 技能库
  □ Hugging Face / Coze / Dify — 智能体工作流
  □ AirTable / Notion — 技能库合集
  □ 关键词："video production skill" / "AI video workflow" / "可灵 workflow"

怎么搜：
  用 MindSearch 或直接搜索引擎
  关键词 = [你的意图] + "Claude Skill" / "AI workflow" / "automation"

搜完之后怎么做：
  ┌─────────────────────────────────────────────┐
  │ 搜到类似的 Skill →                           │
  │                                             │
  │ 学习点1：工具选型                            │
  │   → 调整"叠"的太阳（工具）                  │
  │                                             │
  │ 学习点2：任务流程                            │
  │   → 调整"叠"的少阳（任务顺序）              │
  │                                             │
  │ 学习点3：异常处理                            │
  │   → 补充"展"的坎（异常）                   │
  │                                             │
  │ 学习点4：质量门控                            │
  │   → 调整"叠"的太阴（门控阈值）             │
  └─────────────────────────────────────────────┘

  没搜到类似 Skill → 确认空白，从零推导
  → 但至少知道：同类项目用什么工具/踩过什么坑

搜的产出（填入后续步骤）：
  = 工具候选清单（网上实际用过的）
  = 流程参考模板（别人是怎么做的）
  = 异常处理参考（踩过什么坑）
  = 质量标准参考（行业默认阈值）
```

**搜推导实例：**

```
意图："做一个视频生产 Skill"

搜的过程：
  搜"video production Claude Skill"
  → 搜到 Claude Code 官方的 ai-video-ad-workflow
  → 搜到 Hugging Face 上 MiniMax 的视频生成 Space
  → 搜到可灵 AI 的 API 文档和最佳实践

搜到的关键信息：
  工具：MiniMax 可灵 / Runway / Pika Labs / FFmpeg
  流程：脚本 → 分镜 → 生成 → 后期合成
  踩坑：畸形视频 / 版权音乐 / 时长超限
  质量：封面点击率 / 完播率（但这是后验的，生产阶段只看画面质量）

应用：
  → 太阳（工具）参考可灵 API，而非自己拍脑袋
  → 少阳（任务）参考分镜步骤，加一步"分镜"在脚本之后
  → 坎（异常）参考踩坑记录
  → 太阴（门控）参考行业标准（时长/尺寸）
```

## 一、分：切阴阳

```
太极 → 两仪

把一个意图切成两半：
  阳面：这个 Skill 要做什么
  阴面：这个 Skill 不能做什么

怎么切？

方法：拿意图问自己三个问题

问题1：AI 拿到这个意图，最容易做什么过头的事？
      → 那就是第一条禁止

问题2：这个阶段的数据/决策，属于谁的范围？
      → 那个范围就是第二条禁止

问题3：什么判断必须留给人类？
      → 那就是第三条禁止

问题4（可选）：上下游 Skill 会误以为这个 Skill 也能做？
      → 那就是边界禁止
```

**分推导实例：**

意图："我想分析用户评价"

```
问AI最可能做过头：AI可能会自己修改数据、编造评论、跳过数据直接给结论
→ ✗ 不修改原始数据
→ ✗ 不编造或伪造评价内容
→ ✗ 不跳过数据质量检查直接给结论

问数据属于谁：原始评价数据属于采集阶段
→ ✗ 不做数据采集（那是另一个 Skill 的事）

问什么留给人类：评价的最终判断（这是好评还是差评）可能需要人工复核
→ ✗ 不对单条评价做最终定性判决

阳面：统计频率、找共现模式、生成报告
```

## 二、叠：阴阳叠四力

```
两仪 → 四象

阳面（要做什么） + 阴面（不能做什么） = 叠出四种力量

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
四力          从哪里来              问什么问题
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
太阳（工具）   从"要做什么"里拆出来   我需要什么家伙来做这件事？
少阳（任务）   从工具排序里推出来   先用什么、后用什么？
少阴（判断）   从"不能做什么"里提炼  每一步怎么判断对错？
太阴（门控）   从质量标准里定义     怎么知道这一步做完了？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

怎么叠？

太阳（工具）推导：
  1. 列出阳面每一条（要做什么）
  2. 每一条问：完成它需要什么工具？
  3. 工具合并：同类工具合并，最终清单 3-10 个

少阳（任务）推导：
  1. 从工具清单，问：哪个工具最基础（其他工具依赖它）？
  2. 按依赖关系排顺序
  3. 得到：Step 1 → Step 2 → Step 3 ...

少阴（判断）推导：
  1. 每个任务步骤，问：这个步骤做完，什么情况算对？什么情况算错？
  2. 对的保留，错的下一步处理
  3. 得到：判断1、判断2、判断3 ...

太阴（门控）推导：
  1. 从所有判断里，提炼最终质量标准
  2. 问：这个 Skill 交付时，什么必须有？
  3. 得到：门控清单（通常是 2-4 条）
```

**叠推导实例：**

```
意图："我想分析用户评价"

分的结果：
  阳面 = 统计频率、找共现模式、生成报告
  阴面 = 不修改数据、不编造内容、不跳过检查、不做采集、不做单条定性

叠出四力：

太阳（工具）：
  统计频率 → 计数工具
  找共现 → 共现分析工具
  生成报告 → 报告生成工具
  共 3 个工具

少阳（任务）：
  Step 1: 接收清洗后的数据
  Step 2: 做频率统计
  Step 3: 做共现分析
  Step 4: 生成报告

少阴（判断）：
  判断1：统计结果可信吗？（样本量 >= ___ 条）
  判断2：共现关系有逻辑吗？（至少出现 ___ 次才计入）
  判断3：报告有数据支撑吗？（每个结论 >= 3 条数据）

太阴（门控）：
  □ 统计样本量足够
  □ 每个洞察有数据支撑
  □ 原始数据未被修改（可追溯）
```

## 三、展：四力展八卦

```
四象 → 八卦

四力的每一个，展开成八卦的两个维度：

工具（太阳）展开 → 乾（输入）+ 坤（输出）
任务（少阳）展开 → 震（起势）+ 巽（流向）
判断（少阴）展开 → 坎（异常）+ 离（标准）
门控（太阴）展开 → 艮（边界）+ 兑（握手）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
八卦      从哪个力展开    怎么填
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
乾        太阳            这个 Skill 接收什么数据？
坤        太阳            这个 Skill 产出什么文件？
震        少阳            从哪个任务开始？
巽        少阳            完成后流向哪里？
坎        少阴            每步判断失败时怎么办？
离        少阴            所有步骤完成后，什么状态算成功？
艮        太阴            哪些事绝对不能做？（结合阴面）
兑        太阴            完成后交给谁，怎么交？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**展推导实例：**

```
意图："我想分析用户评价"

从分和叠的结果展开八卦：

乾（输入，太阳展开）：
  = data-etl 输出的清洗后评价数据文件路径
  = 评价字段：平台、用户名、时间戳、评价内容、评分、点赞数

坤（输出，太阳展开）：
  = 统计报告（JSON）：频率统计 + 共现矩阵
  = 洞察报告（Markdown）：痛点/爽点/机会清单

震（起势，少阳展开）：
  = Step 1: 读取数据文件，开始频率统计

巽（流向，少阳展开）：
  = 输出流向 voc-insight（统计报告 + 洞察报告）

坎（异常，少阴展开）：
  = 判断1失败（样本不足）→ 停止，输出"样本不足，无法分析"
  = 判断2失败（共现无逻辑）→ 跳过共现分析，只做频率统计
  = 判断3失败（无数据支撑）→ 报告标注"结论待验证"

离（标准，少阴展开）：
  = 统计报告和洞察报告都已生成
  = 每条洞察有数据证据
  = 无原始数据被修改

艮（边界，太阴展开）：
  （从分的结果——阴面）
  = 不修改原始数据
  = 不采集新数据
  = 不对单条评价做定性判决

兑（握手，太阴展开）：
  = 文件输出到约定目录
  = voc-insight 通过读取该目录触发
  = 附带元信息：处理时间、样本量、洞察数量
```

## 四、衡：五行动态推理

```
八卦 → 五行

八卦填完之后，用五行做动态推理。

五行不是5个检查项，是5种力量。
力量之间有生有克，构成活的推理链条。

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
力量        代表什么          不是静态定义
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
木          发展方向/意图     不是"任务"这个字
火          判断力/决策力     不是"质量门控"这个字
土          沉淀/产出/结果   不是"位置"这个字
金          工具/结构/边界   不是"工具清单"
水          流动/传承/连接   不是"流向"这个字
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

生克关系 = 活的推理链条，不是检查清单：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
相生：力量从哪来
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
木 → 火    有方向才有判断
火 → 土    有判断才有沉淀
土 → 金    有沉淀才有结构
金 → 水    有结构才有流动
水 → 木    有流动才有新方向

相克：力量往哪去
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
金 → 木    结构修剪方向（太散的方向被约束）
木 → 土    创新突破沉淀（太稳的结构被打破）
土 → 水    沉淀阻挡流动（太多沉淀堵住）
水 → 火    流动平衡冲动（太热的判断被冷却）
火 → 金    决策瓦解结构（太硬的结构被调整）
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**五行推理的正确方式：不是检查，是追问**

```
不是问："工具够吗？任务有序吗？判断有阈值吗？"

而是问：

木（方向）→ 这个 Skill 的核心发展方向是什么？
  → 推动它的力量是谁（水）？是否充沛？
  → 其判断力是否充沛（火充沛与否）？
  → 其方向是否过度发散（木被金克过度）？

火（判断）→ 这个 Skill 的判断力来自哪里？
  → 判断是否有方向支撑（木生火）？
  → 判断是否产生沉淀（火生土）？
  → 判断是否过度僵化（火被水克过度）？

土（沉淀）→ 这个 Skill 产出了什么？
  → 产出是否形成结构（土生金）？
  → 产出是否过度冗余/无序（木克土过度）？

金（结构）→ 这个 Skill 的工具有没有边界？
  → 工具是否充沛（金生水）？
  → 工具是否过度僵化（火克金过度）？

水（流动）→ 这个 Skill 的上下游是怎么连接的？
  → 连接是否有方向（水生木）？
  → 连接是否过度发散/无序（土克水过度）？
```

**生克断裂的症状（不是报错，是失衡）：**

```
木生火 断裂：
  → 有意图但不知道怎么做（方向有，判断力没有）
  → 火弱：判断力不足，阈值模糊

火生土 断裂：
  → 一直在判断但没有产出（判断多，沉淀没有）
  → 土弱：输出不明确，报告质量低

土生金 断裂：
  → 有产出但没有形成规范（结果有，结构没有）
  → 金弱：工具不统一，流程碎片化

金生水 断裂：
  → 有结构但流动不起来（工具有，任务流没有）
  → 水弱：上下游不通，输入输出对不上

水生木 断裂：
  → 流动了但没有新方向（流了，但没有生长）
  → 木弱：没有扩展预设，Skill 一旦定型就死

金克木 断裂：
  → 方向太散无法执行（结构太弱约束不住方向）
  → 木旺金弱：想做太多，工具跟不上

火克金 断裂：
  → 结构太死无法调整（判断太硬，结构改不动）
  → 金旺火弱：标准定死了，异常无法处理
```

**五行推理模板（填完八卦后用）：**

```
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
木（方向）：
  这个 Skill 要去哪？→ 扩展预设的方向是什么？
  推动它的力量（水）→ 上游从哪来？是否充沛？
  其判断力（火）是否充沛？→ 是否过度发散？

火（判断）：
  判断的依据是什么？→ 判断力从哪来（木生火了吗）？
  判断产生了什么（土）？→ 有沉淀吗？
  判断是否过度僵化？→ 水克火了吗？流动是否达到平衡？

土（沉淀）：
  沉淀是什么？→ 输出文件/洞察/报告
  沉淀变成结构了吗（金）？→ 工具规范了吗？
  沉淀是否阻塞？→ 木克土了吗？是否有创新突破？

金（结构）：
  工具的边界是什么？→ 禁止规则
  结构允许流动吗（水）？→ 上下游是否贯通？
  结构会限制生长吗？→ 火克金了吗？是否过度僵化？

水（流动）：
  上下游怎么连的？→ 巽（流向）+ 兑（握手）
  流向了新方向吗（木）？→ 有扩展吗？
  是否过度发散/是否阻塞？→ 土克水了吗？沉淀太多了吗？
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**五行完整实例（竞品价格监控）：**

```
木（方向）：
  Skill 的方向：监控竞品 → 支撑品牌战略
  水生木：品牌战略作为上游，给出监控方向 ✓
  火充沛性：告警阈值 10% → 判断力清晰 ✓
  木被金克：工具精度够（能解析动态页面）✓

火（判断）：
  判断力从木来：采集→比对→告警，方向清晰 ✓
  火生土：判断产生沉淀 → daily_report + alert ✓
  土充沛性：有日报和告警两个产出 ✓
  水克火：告警去重机制✓

土（沉淀）：
  沉淀是：日报 + 告警
  土生金：沉淀变成规范 → 输出格式固定 ✓
```

## 五、验：失衡回头补

```
五行推理中发现失衡，用验回头补。

不是查清单，是判断五种力量的失衡症状：

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
失衡症状              五行根源       回头补
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
想做太多但做不了      木旺金弱       叠（太阳）加强边界
一直在做但没产出      火旺土弱       叠（少阴）加强判断标准
有产出但很散乱        土旺水弱       展（坤）明确输出格式
输出对了但上下游不通    金旺水弱       展（巽/兑）定义握手协议
上下游通了但没有方向    水旺木弱       验（木）预设扩展方向

一直在判断但不能决策    火被水克太过   叠（太阴）收紧阈值
结构过度僵化无法调整    金被火克太过   叠（太阳）放松禁止边界
方向过度发散无法执行    木被金克太过   衡（木）聚焦扩展方向
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

**验的完整流程（不是检查清单，是动态追问）：**

```
1. 问木（方向）：
   → 有方向吗？清晰吗？（方向不清 → 木弱）
   → 扩展预设是否已定义？（没有扩展预设 = 木死）

2. 问火（判断）：
   → 判断有依据吗？依据从哪来？（无依据 = 火弱）
   → 判断是否过度僵化/松散？（过度僵化 = 火克金太过 / 过度松散 = 水克火太过）

3. 问土（沉淀）：
   → 有产出吗？产出清晰吗？（无产出 = 土弱）
   → 产出是否过度冗余/无序？（过度无序 = 土被木克太过）

4. 问金（结构）：
   → 工具边界清晰吗？边界是否过度僵化/松散？（过度僵化 = 火克金太过 / 过度松散 = 金克木太弱）

5. 问水（流动）：
   → 上下游贯通了吗？流向是否正确？（未贯通 = 金生水断）
   → 流是否阻塞/过度发散？（阻塞 = 土克水太过 / 过度发散 = 水生木太弱）

扩展性验证（每个 Skill 必须回答）：
  □ 木（方向）已定义？→ 未定义 = 木死，Skill 一旦定型就死
  □ 木的扩展方向是什么？→ 扩展方向：___ / ___ / ___
  □ 扩展规则是什么？
    → 轻量扩（只填八卦）→ 重量扩（回到叠）→ 架构扩（回到衡）
  □ 扩展门控是什么？
    → 新工具不冲突 / 新任务兼容 / 新输出对上下游兼容
    → 扩展门控：___ / ___ / ___
```

**验通过的标准：**

```
不是"没有悬空工具"这种检查项

而是：
✓ 木（方向）清晰，有扩展预设
✓ 火（判断）有依据，判断充分
✓ 土（沉淀）有产出，产出适量
✓ 金（结构）有边界，边界清晰
✓ 水（流动）流向贯通，流向明确

五行生克顺畅，力量平衡
→ 验通过
```
