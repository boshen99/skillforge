## Entry 1: Chain Node

### When to use

```
Your Skill is part of:
  VOC chain: data-collection вҶ’ data-etl вҶ’ pattern-finder вҶ’ voc-insight
  Or: extends a specific node (adds a sub-node)

Does it have clear upstream and downstream?
  Yes вҶ’ Chain Node вң“
  No  вҶ’ Might be independent вҶ’ go to Entry 2
```

### Chain Node вҖ” Split questions (specific)

```
In addition to the three universal questions, also ask:

Q-L1: What data will upstream give me?
       вҶ’ Most of Qian (Input) comes from here

Q-L2: What data does downstream need from me?
       вҶ’ Most of Kun (Output) goes here

Q-L3: What would upstream/downstream most mistakenly think I can also do?
       вҶ’ Key Yin rules for Gen (Boundary) grow from here

Q-L4: What is the data format of upstream/downstream?
       вҶ’ Defines Xun (Flow) and Dui (Handshake) protocols
```

### Chain Node вҖ” Stack questions (specific)

```
Sun (Tools):
  Upstream uses what tools? вҶ’ Do I need to be compatible?
  Downstream needs what tools? вҶ’ Do I need to output in a format it can read?

Young Yang (Task Order):
  Do upstream/downstream Steps align?
  вҶ’ Does my Step 1 correspond to which upstream output?
  вҶ’ Does my last step correspond to which downstream input?

Kan (Risk):
  Upstream data fails? вҶ’ Do I tolerate bad input or reject it?
  Downstream can't receive? вҶ’ Do I wait or alert?
```

```

```

## Entry 3: Strategy Chain

### When to use

```
Your Skill is in:
  L1 Understand вҖ” Where am I? (user insights / market analysis / category assessment)
  L2 Change вҖ” Where am I going? (reposition / validate hypotheses)
  L3 Hold вҖ” How do I keep it? (brand building / execution)

Are you making decisions or analyzing data?
  Decision-driven вҶ’ Strategy Chain вң“
  Data-driven вҶ’ Might not be strategy вҶ’ go to Entry 1 or 4
```

### Strategy вҖ” Split questions (specific)

```
Q-S1: What can this level (L1/L2/L3) NOT skip to?
  L1: Cannot give execution advice (still in understanding phase)
  L2: Cannot skip validation and go straight to conclusion
  L3: Cannot give strategic options (already in execution phase)

Q-S2: What decisions must be left to humans?
  Strategic direction / resource allocation / make-or-break choices

Q-S3: Can your hypothesis be falsified?
  Yes вҶ’ can enter L2  No вҶ’ still in L1
```

### Strategy вҖ” Stack questions (specific)

```
L1 (Understand) Four Forces:
  Sun: interview tool, analysis tool, mapping tool
  Young Yang: collect insights вҶ’ categorize вҶ’ form understanding
  Young Yin: Does user really think this? Where's the evidence? Any counter-examples?
  Dark: в–Ў data-supported  в–Ў has categorization  в–Ў has priority

L2 (Change) Four Forces:
  Sun: hypothesis tool, validation tool, decision tool
  Young Yang: propose hypothesis вҶ’ design validation вҶ’ confirm direction
  Young Yin: Is hypothesis bold enough? Can it be falsified? Is evidence sufficient?
  Dark: в–Ў hypothesis clear  в–Ў falsifiable  в–Ў conclusions actionable

L3 (Hold) Four Forces:
  Sun: execution tool, monitoring tool, evaluation tool
  Young Yang: plan вҶ’ allocate resources вҶ’ execute
  Young Yin: Can plan be executed? Resources sufficient? Milestones defined?
  Dark: в–Ў has execution plan  в–Ў has milestones  в–Ў has evaluation metrics
```

## Entry 4: Analytics & Insight

### When to use

```
Your Skill's core is two layers:
  Layer 1: Statistics (what is happening)
  Layer 2: Insight (why is it happening)

Is the user asking "how many" or "why"?
  First ask how many вҶ’ Analytics & Insight вң“
  Only ask what's there вҶ’ might be collection/cleaning вҶ’ go to Entry 1
```

### Analytics & Insight вҖ” Stack questions (specific)

```
Statistics Layer (Sun + Young Yang):
  Sun tools: frequency counter, co-occurrence analyzer, trend tool, comparison tool
  Young Yang: Step1 frequency вҶ’ Step2 co-occurrence вҶ’ Step3 trend/comparison

Insight Layer (Sun + Young Yang):
  Sun tools: root cause excavator, opportunity identifier, strategy generator
  Young Yang: Step4 from stats вҶ’ hypothesize
              Step5 validate вҶ’ find evidence
              Step6 form strategy вҶ’ output recommendations

Core: Statistics conclusions вҶ’ Insight layer inputs
  Not two separate Skills вҖ” one Skill with two-layer architecture
```

### Analytics & Insight вҖ” Young Yin (judgment) questions

```
J-H1: Is the statistical conclusion credible?
      вҶ’ Sample size >= ___ records
      вҶ’ Data sources traceable

J-H2: Does the insight have evidence?
      вҶ’ Each insight >= 3 data points
      вҶ’ No counter-examples (or counter-examples explained)

J-H3: Is the insight actionable?
      вҶ’ Not generic talk
      вҶ’ Has specific recommendation/direction

J-H4: Is the hypothesis bold enough?
      вҶ’ Not an obvious fact
      вҶ’ Has insight delta (something user doesn't know)
```
